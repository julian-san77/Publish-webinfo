Project Web/Plugin

1
Jangan hapus file yang ada didalam folder tanpa sepengetahuan user
2
Ketika melakukan perubahan atau penambahan fitur, pastikan untuk memberikan dokumentasi di dalam dev_log.md
3
Fitur update harus terdokumentasi dengan jelas setelah melakukan perubahan fitur
4
Pastikan bahwa menjelaskan plan atau rencana perubahan fitur dengan jelas sebelum melakukan perubahan kode
5
Jangan menghapus kode yang berfungsi dengan baik, jika ingin menambahkan silahkan tambahkan kode baru saja tanpa menghapus kode yang ada, jika memang bisa
6
Untuk awal selalu buat file bat dalam folder project yang berfungsi membuat folder dan mengcopy file ke dalam folder bernama "backup"
7
Gunakan file dalam folder backup sebagai cara revert back/rolling back jika terjadi corrupt pada file

Pembuatan Blog

1
Membuat blog harus terdokumentasi dengan jelas setelah melakukan perubahan fitur
2
Mendapatkan informasi yang dibutuhkan melalui website yang terpercaya
3
User akan memberikan informasi yang akan dibutuhkan oleh blog, gunakan itu sebagai isi blog
